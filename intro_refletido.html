XSS Refletido

No ataque XSS refletido, o atacante gera um (URL) que contém um código malicioso, como parte da requisição HTTP. O código malicioso é inserido em campos de entrada da aplicação Web, como barras de busca, seções de comentários e caixas de texto de autenticação.

Ao receber um URL da página Web servida e executá-la em seu navegador, o usuário legítimo executa um código malicioso presente no URL que foi enviado pelo atacante. Nesse sentido, o usuário acredita estar acessando a página Web em seu formato legítimo, contudo, o que o usuário irá visualizar pode variar de acordo com a natureza do ataque.

Dependendo do nível de sofisticação, pode ser que a visualização apenas seja modificada com a adição de janelas de pop-ups falsas, formulários de dados ou links de anúncios fraudulentos, já em ataques mais elaborados, é possível que o atacante crie uma página Web completamente diferente (contendo uma identidade visual idêntica à página legítima) e adicione um redirecionamento da página em questão para a página fraudulenta criada, adicionando elementos como formulários que capturam as informações adicionadas pelo usuário e as enviam para um servidor remoto a fim de que o atacante tenha acesso.
