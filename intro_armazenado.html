XSS Armazenado

No XSS armazenado, o atacante gera um código malicioso que será armazenado persistentemente em um servidor vulnerável, geralmente através de um banco de dados. Um exemplo deste tipo de ataque pode ser encontrado em aplicações Web que servem como fóruns de discussão ou redes sociais.

Este tipo de ataque não necessita um URL para que seja executado e pode afetar um número maior de usuários, visto que qualquer usuário que navegue pela aplicação Web pode executar o código malicioso servido pela página comprometida. Este tipo de ataque pode não só ter o objetivo de prejudicar o usuário legítimo que está navegando pela página Web, mas também, de alguma forma, prejudicar o servidor Web e as aplicações que estão mantendo a determinada página ativa.

Além de o código malicioso inserido na página poder ser utilizado para tentar coletar informações dos usuários legítimos que a estão acessando ou redirecionar estes usuários para outra página Web criada pelo atacante, também é possível que o código tenha como objetivo coletar informações armazenadas no servidor que hospeda a página, como informações de usuários cadastrados na página ou informações de login e senha para acesso administrativo ao servidor.
